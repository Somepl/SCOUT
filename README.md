# SCOUT - 行情态势感知系统

**Signal Capture, Observation, Understanding & Track**

面向 A 股中长线投资者的情报分析系统。每天自动采集财经新闻，AI 分析后生成投资建议简报、个股交易仪表盘、狙击清单，并通过 Web 看板可视化呈现。

## 核心模块与实现原理

### 1. 采集层 — collector.py

**原理**: 对每个信息源使用 HTTP GET 请求获取页面 HTML，用 BeautifulSoup + lxml 解析 DOM 树提取新闻标题、内容、发布时间。

**4 个信息源解析器**:

| 解析器 | 目标 | 选择器策略 |
|--------|------|-----------|
| `parse_eastmoney` | 东方财富要闻 | `.newsItem` → `h3 a` 提取标题+链接，跳转详情页获取正文 |
| `parse_sina` | 新浪财经头条 | `.blk12 li` → 提取标题+摘要，过滤广告/App推广 |
| `parse_10jqka` | 同花顺要闻 | `.news_list li` → 提取标题+时间 |
| `parse_stcn` | 证券时报 | `.news_list li` → 提取标题+摘要 |

**容错机制**: 每个解析器独立 try/except，单个源失败不影响其他源。

### 2. AI 分析层 — analyzer.py

#### 2.1 使用的模型

| 模型 | 提供商 | 费用 | 用途 |
|------|--------|------|------|
| DeepSeek V4 Flash | DeepSeek 官方 | ¥0（500万免费额度） | 新闻分析 + 个股仪表盘 |
| 可切换任意 OpenAI 兼容模型 | 用户自选 | 视模型而定 | 在 config.py 改三行即可 |

#### 2.2 新闻分析 — `analyze_news_batch()`

**输入**: 新闻标题 + 来源 + 内容摘要  
**输出**: 结构化分析结果，包含以下字段：

| 字段 | 类型 | 可能取值 | 判定逻辑 |
|------|------|---------|---------|
| `event_type` | str | 政策利好/政策利空/行业利好/行业利空/公司利好/公司利空/外围市场/中性消息 | AI 根据新闻内容分类 |
| `confidence` | str | 高/中/低 | AI 评估消息来源可靠性、信息明确度 |
| `reason` | str | 自由文本 | AI 给出判断依据 |
| `affected_sectors` | list | 如 ["锂电池", "新能源汽车"] | AI 提取新闻影响的行业板块 |
| `stock_codes` | list | 如 ["300750", "600519"] | AI 从新闻中提取明确提到的 A 股代码 |
| `impact_level` | str | 重大/中等/轻微 | AI 评估事件对市场的影响程度 |
| `time_horizon` | str | 短期/中期/长期 | AI 判断影响持续时间 |
| `market_sentiment` | str | 利好/利空/中性 | AI 综合判断对市场的影响方向 |
| `advice` | str | 买入/观望/卖出/关注 | AI 给出的投资操作建议 |
| `advice_reason` | str | 自由文本 | AI 解释建议理由 |

**提示词策略**: 使用 System Prompt + User Template 结构，要求 AI 以 KV 格式或 JSON 格式输出。优先尝试 JSON 解析，失败时回退到 KV 格式解析。

**重试机制**: API 调用失败时自动重试 2 次，间隔 2 秒。

#### 2.3 个股交易仪表盘 — `analyze_stock_dashboard()`

**输入**: 个股行情数据（价格、均线、MACD、RSI、量比、支撑阻力等）  
**输出**: 完整的交易决策 JSON，包含三个部分：

**a) core_conclusion（核心结论）**
- `signal`: 强烈买入 / 买入 / 持有 / 观望 / 卖出 / 强烈卖出
- `confidence`: 高/中/低
- `trading_advisor_rating`: ★★★★★ ~ ★☆☆☆☆
- `summary`: 一句话核心结论
- `risk_warning`: 风险提示

**b) data_perspective（数据视角）**
- `trend`: 多头/空头/震荡 + 强度 + 均线排列 + 乖离率
- `volume`: 放量上涨/缩量回调/放量下跌/缩量上涨/量能正常 + 量比
- `macd`: 零轴上金叉/金叉/多头/死叉/空头/中性 + DIF/DEA/BAR
- `rsi`: RSI6/12/24 + 超买/强势买入/中性/弱势/超卖
- `support_resistance`: 支撑位列表 + 阻力位列表

**c) battle_plan（作战计划）**
- `action`: 买入/加仓/持有/减仓/卖出/观望
- `entry_plan`: 理想买点 + 次优买点 + 止损位
- `take_profit`: TP1/TP2/TP3 止盈目标
- `position`: 建议仓位 + 分仓策略
- `risk_control`: 最大亏损 + 风险清单

**决策稳定化** (`_stabilize_decision`): 当 AI 输出的 action 与 signal 矛盾时自动修正。例如 action="买入" 但 signal="观望" 时降级为"观望"。买入标的必须设置止损和止盈。

### 3. 量化评分模型 — quant_model.py

用 LightGBM 替代固定规则评分，从历史数据中学习什么特征组合能预测涨跌。

**特征工程**: 从 K 线数据计算 61 个量化特征，分 6 大类：

| 类别 | 特征数 | 示例 |
|------|--------|------|
| 收益率 | 10 | ret_5d、ret_std_10d |
| 价格位置 | 9 | pos_in_20d、pct_off_high |
| 均线 | 12 | ma_10、dist_ma_20、ma_slope |
| 成交量 | 6 | vol_ratio_5d、vol_trend |
| 技术指标 | 6 | RSI6/12/24、MACD |
| 形态 | 4 | upper_shadow、body_pct |

**训练方法**: 对每只股票逐日计算特征，目标值为未来 5 日收益率。所有股票数据合并后训练单一 LightGBM 回归模型。

**评分融合**: `最终评分 = 规则评分 × 50% + 模型评分 × 50%`

**验证结果**（150只股票×800天，57,450条样本）:
- 验证 MAE: ±3.50%（预测 5 日收益的平均误差）
- 相关系数: 0.482
- 方向准确率: 61.4%（涨/跌分类）
- 分类辅助模型准确率: 69.6%
- 最优特征（含日历特征）: day_of_month、month、vol_trend_20d、lower_shadow

### 4. 行情技术分析 — market.py

#### 3.1 数据来源
- **实时行情**: 新浪财经 API (`hq.sinajs.cn`)
- **K 线数据**: 新浪财经 K 线 API (`money.finance.sina.com.cn/quotes_service/api/json_v2.php`)
  - 获取最近 60 个交易日的日 K 线

#### 3.2 技术指标计算方法

| 指标 | 计算方法 | 参数 |
|------|---------|------|
| **MA(均线)** | 简单移动平均 | 5/10/20/60 日 |
| **MACD** | EMA(12) - EMA(26) 为 DIF DEA = EMA(DIF, 9) BAR = (DIF - DEA) × 2 | 快线12, 慢线26, 信号9 |
| **RSI** | RSI = 100 - 100/(1 + RS) RS = 平均涨幅/平均跌幅 使用 EMA 平滑 | 6/12/24 周期 |
| **乖离率** | (价格 - 均线) / 均线 × 100% | MA5/MA10/MA20 |
| **量比** | 当日成交量 / 近5日平均成交量 | 5日基准 |

#### 3.3 综合评分系统 — `_generate_signal()`

评分由多个维度加权求和，总分 0-100：

| 维度 | 最高分 | 判定逻辑 |
|------|--------|---------|
| **趋势** | 30分 | 强势多头(30) > 多头排列(26) > 弱势多头(18) > 盘整(12) > 弱势空头(8) > 空头排列(4) > 强势空头(0) |
| **乖离率** | 20分 | 价格回踩MA5(-3%~0%) 最佳(20)；略高于MA5(0%~2%) (18)；乖离过大 >7.5% 扣分(4) |
| **量能** | 15分 | 缩量回调(15) > 放量上涨(12) > 量能正常(10) > 缩量上涨(6) > 放量下跌(0) |
| **MACD** | 15分 | 零轴上金叉(15) > 金叉(12) > 上穿零轴(10) > 多头(8) > 空头(2) > 死叉/下穿零轴(0) |
| **RSI** | 10分 | 超卖(10) > 强势买入(8) > 中性(5) > 弱势(3) > 超买(0) |
| **均线支撑** | 10分 | MA5 支撑有效(+5) + MA10 支撑有效(+5) |

**最终信号判定**:
| 评分 | 趋势条件 | 信号 |
|------|---------|------|
| ≥75 | 强势多头/多头排列 | 强烈买入 |
| ≥60 | 多头或弱势多头 | 买入 |
| ≥45 | 不限 | 持有 |
| ≥30 | 不限 | 观望 |
| <30 | 空头排列/强势空头 | 强烈卖出 |
| 其他 | | 卖出 |

### 5. 策略生成层

#### 4.1 热点筛选 — screener.py

**原理**: 从新闻分析结果中提取热点板块，匹配预定义的板块-股票映射表，结合 AI 推荐生成候选股票池。

**流程**:
```
新闻分析结果
  → 提取每条新闻的 affected_sectors
  → 统计板块提及次数 + 情感得分（利好加分/利空减分）
  → 排序得到热点板块列表
  → 从板块映射表（80+板块, 400+股票）取出对应股票
  → 调用 AI 根据新闻内容额外推荐股票
  → 合并去重得到最终候选池
```

**板块映射表**: 涵盖 50+ 个 A 股热门板块（白酒、银行、新能源汽车、半导体、AI、军工等），每个板块预设 3-5 只代表股票。

**情感加权算法**: 
- `weight = confidence_score × impact_score`（高=3, 中=2, 低=1）×（重大=3, 中等=2, 轻微=1）
- 利好新闻加正分，利空新闻加负分
- 最终按净情感分排序，只保留利好板块的股票

#### 4.2 狙击清单 — strategist.py

**原理**: 对个股分析结果进行二次评分排序，筛选出可执行的买入标的，分配仓位。

**二次评分算法**:
```
final_score = base_score(技术评分)
  + volume_bonus(缩量回调或放量上涨 +5)
  + trend_strength × 0.1
  - volatility_penalty(乖离MA5>7% -10, 乖离MA20>10% -10)
```

**仓位分配**:
| 排名 | 仓位比例 |
|------|---------|
| 1 | 35% |
| 2 | 25% |
| 3 | 20% |
| 4 | 12% |
| 5 | 8% |
| 6+ | 5% |

**过滤条件**: 只有 action="买入/加仓" 或 signal="强烈买入/买入" 且技术评分 ≥ 60 的股票才会进入清单。

#### 4.3 资金面分析 — capital.py

**数据源**:
- **北向资金流向**: 东方财富 API (`push2.eastmoney.com/api/qt/kamt.kline/get`)
- **融资融券余额**: 东方财富数据中心 API
- **沪深港通十大成交**: 东方财富 API

**资金面信号算法** (`calc_capital_light`):
- 取最近 3 个交易日北向资金流向
- 计算净流入天数比例 + 平均净流入额
- 比例 ≥ 66% 且平均为正 → 利好(80分)
- 比例 ≥ 50% 且平均为正 → 偏积极(65分)
- 比例 ≤ 33% 且平均为负 → 利空(20分)
- 比例 < 50% 且平均为负 → 偏谨慎(35分)
- 其他 → 中性(50分)

### 6. 市场综合信号 — reporter.py

#### 市场光信号 `calc_market_light()`

综合新闻面和技术面的多维度评分系统：

**新闻面评分** (权重 ±20):
- 利好/利空数量比 ≥ 3 → +20（强烈偏多）
- 利好/利空数量比 ≥ 2 → +15（偏多）
- 利好略多于利空 → +8（略偏多）
- 反之对应扣分

**技术面评分** (权重 ±10):
- 个股平均评分 ≥ 65 → +10（技术偏多）
- 个股平均评分 ≥ 50 → +5（技术中性）
- 个股平均评分 < 50 → -8（技术偏空）

**最终信号**:
| 总分 | 显示 | 含义 |
|------|------|------|
| ≥75 | 🟢🟢🟢 | 情绪乐观，可积极参与 |
| ≥60 | 🟢🟢⚪ | 情绪偏暖，精选个股 |
| ≥40 | 🟢⚪⚪ | 情绪中性，谨慎参与 |
| ≥25 | ⚪⚪🔴 | 情绪偏冷，控制仓位 |
| <25 | 🔴🔴🔴 | 情绪悲观，建议空仓 |

## 完整数据流 (11步全链路)

```
Step 1:  采集 → collector.py 从4个源获取原始新闻
Step 2:  去重 → utils.deduplicate MD5去重
Step 3:  分析 → analyzer.analyze_news_batch AI逐条分析
Step 4:  存储 → storage.save_analysis_batch 写入SQLite
Step 5:  简报 → reporter.build_report 生成新闻简报
Step 6:  筛选 → screener.discover_stocks 发现候选股票
Step 7:  行情 → market.get_stock_analysis + analyzer.analyze_stocks_batch
Step 8:  资金 → capital.get_capital_summary + reporter.build_capital_report
Step 9:  存储个股 → storage.save_stock_analysis_batch 写入SQLite stock_analysis表
Step 10: 回测 → backtest.run_backtest 验证历史买入建议有效性
Step 11: 推送 → notifier.push_wechat 微信推送 + 文件存档
```
